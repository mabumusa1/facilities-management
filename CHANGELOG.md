### Added
- Reports — occupancy report: per-community unit counts by status (occupied, available, maintenance, off-plan) with occupancy rate percentage ([#382](https://github.com/mabumusa1/facilities-management/pull/382))
- Reports — lease pipeline: expiring leases grouped by month with renewal counts; configurable look-ahead window up to 24 months ([#382](https://github.com/mabumusa1/facilities-management/pull/382))
- Reports — portfolio health: community-level breakdown of occupied, under-maintenance, and off-plan units with occupancy rates ([#382](https://github.com/mabumusa1/facilities-management/pull/382))
- Reports — financial summary: revenue, collected, and outstanding amounts by period with collection rate percentage ([#382](https://github.com/mabumusa1/facilities-management/pull/382))
- Accounting — bank account management: add, update, and delete bank accounts per community; IBAN, account number, and currency fields; single-default enforcement across all accounts ([#380](https://github.com/mabumusa1/facilities-management/pull/380))
- Accounting — payment recording: record full and partial payments against transactions with payment method, reference, and notes; auto-updates transaction paid status when fully settled ([#380](https://github.com/mabumusa1/facilities-management/pull/380))
- Accounting — aging report: overdue receivables grouped by 0/1-30/31-60/61-90/90+ day buckets with count and total per bucket; configurable as-of date ([#380](https://github.com/mabumusa1/facilities-management/pull/380))
- Accounting — reconciliation: mark transactions as reconciled with timestamp and actor audit trail; prevents double-reconcile; reconciliation summary shows total/reconciled/unreconciled counts ([#380](https://github.com/mabumusa1/facilities-management/pull/380))
- Accounting — financial summary: income, expense, and net totals with date-range filter per contact, unit, or community ([#380](https://github.com/mabumusa1/facilities-management/pull/380))

- Facilities → Calendar — admins can view all bookings across all facilities on a weekly calendar grid, filter by facility or status (Confirmed / Checked-in / Completed / Cancelled), navigate weeks, and click any booking block to open a detail popover with check-in and cancel actions; admins can also create bookings on behalf of residents from an empty time slot or the **+ Create Booking** button, with overlap detection preventing double-booking ([#376](https://github.com/mabumusa1/facilities-management/pull/376))
- Leasing — lease application approval workflow: managers with community scope can approve or reject a Pending Application lease from the lease detail page; approve transitions the lease to Approved and records the approver identity and timestamp; reject opens a dialog requiring a reason (min 10 / max 2 000 chars), transitions to Rejected with the reason recorded; the Approval Timeline card shows the full audit of approval/rejection events with actor name and timestamp; cross-tenant isolation enforced — managers from other communities receive 403 ([#377](https://github.com/mabumusa1/facilities-management/pull/377))
- Service Requests — admins can triage all incoming requests from **Services → Service Requests**: four-tab queue (All / Unassigned / Overdue / SLA Breach) with filters by status, category, community, and urgency; overdue rows highlighted in red; request detail page shows resident info, location, category, urgency, and description; admins assign a technician (tenant-scoped dropdown) and set priority (Low / Medium / High / Urgent) then click **Assign & Save**; internal notes thread on the detail page is admin-only and never visible to residents; access is gated to Admin and Manager roles (residents receive 403) ([#375](https://github.com/mabumusa1/facilities-management/pull/375))
- Facilities — admin calendar view: browse all bookings in a date range filtered by facility or community; prevents double-booking with overlapping slot detection ([#378](https://github.com/mabumusa1/facilities-management/pull/378))
- Facilities — booking check-in/check-out: mark a reservation as arrived or departed; records timestamps and officer identity; checkout auto-notifies the next person on the waitlist when a slot frees up ([#378](https://github.com/mabumusa1/facilities-management/pull/378))
- Facilities — waitlist management: join a queue for a full facility slot with date and time selection; leave the waitlist at any time; auto-notification when a slot opens via checkout ([#378](https://github.com/mabumusa1/facilities-management/pull/378))
- Facilities — operational reporting: utilisation rate, no-show rate, cancelled count, and peak booking hours per facility or community; date-range filterable ([#378](https://github.com/mabumusa1/facilities-management/pull/378))
- Visitor Access — gate check-in: scan QR code or enter code manually to admit a pre-registered visitor; logs entry timestamp, gate officer identity, and ID verification status; enforces QR expiry, cancellation, double check-in prevention, and max scans per invitation (per-community configurable) ([#373](https://github.com/mabumusa1/facilities-management/pull/373))
- Visitor Access — gate check-out: mark a visitor as departed with one tap; logs exit timestamp and calculates visit duration; blocks duplicate check-out attempts ([#373](https://github.com/mabumusa1/facilities-management/pull/373))
- Visitor Access — walk-in admission: admit unregistered visitors with manual name/phone entry when the community's allow_walk_in setting is enabled; creates a log entry without requiring a QR invitation ([#373](https://github.com/mabumusa1/facilities-management/pull/373))
- Visitor Access — gate officer today view: paginated list of today's expected visitors searchable by name, phone, or QR token; shows check-in/check-out status per visitor ([#373](https://github.com/mabumusa1/facilities-management/pull/373))
- Visitor Access — overstay detection: flag visitors who have not checked out beyond a configurable hour threshold (default 24 h); dedicated overstay report endpoint returns all currently-still-inside visitors with hours-since-entry ([#373](https://github.com/mabumusa1/facilities-management/pull/373))
- Visitor Access — visitor log reporting: full audit trail with date-range, community, and name filters; shows entry/exit times, duration, gate officer, ID verification status, and overstay flag ([#373](https://github.com/mabumusa1/facilities-management/pull/373))
- Service Requests — SLA tracking: automatic breach detection for response and resolution deadlines; per-request SLA_due timestamps calculated from category response_hours and resolution_hours; nightly `checkAll` job flags breaches; check-sla endpoint for on-demand verification ([#370](https://github.com/mabumusa1/facilities-management/pull/370))
- Service Requests — resident rating: 1–5 star rating with optional text feedback on completed requests; validation enforces rating range and 1 000-char feedback limit; rating and feedback persisted on the request record ([#370](https://github.com/mabumusa1/facilities-management/pull/370))
- Service Requests — complaint conversion: admin can convert a Complaint into a Service Request with a category assignment; the complaint status auto-transitions to converted_to_sr and the SR records the source_complaint_id for back-reference ([#370](https://github.com/mabumusa1/facilities-management/pull/370))
- Contacts — KYC document management: polymorphic ContactDocument model stores government ID, passport, licence, and other verification documents on Resident, Owner, or Professional contacts with type classification and expiry tracking ([#369](https://github.com/mabumusa1/facilities-management/pull/369))
- Contacts — unit ownership assignment: UnitOwnership links owners to units with ownership type (full/partial), percentage, and date range; unique constraint prevents duplicate owner-unit pairs; ownedUnits BelongsToMany pivot for listing an owner's portfolio ([#369](https://github.com/mabumusa1/facilities-management/pull/369))
- Contacts — 360-degree activity history: ContactActivity logs key events (lease created, payment received, SR assigned) on every contact type via polymorphic relation; metadata JSON captures event-specific details for audit and timeline views ([#369](https://github.com/mabumusa1/facilities-management/pull/369))
- Contacts — soft delete and archive: Resident, Owner, and Professional contacts now support soft deletes with a status column (active/archived); deleted contacts are recoverable without data loss ([#369](https://github.com/mabumusa1/facilities-management/pull/369))
- Settings — contract type management: create, edit, delete contract types (Yearly Rental, Commercial, etc.) with bilingual EN/AR names, default payment terms, and active/inactive status; full CRUD API consumed by Leasing quote creation ([#366](https://github.com/mabumusa1/facilities-management/pull/366))
- Settings — app appearance / white-label: upload portal logo and favicon, set sidebar primary colour, override navigation labels per tenant from a single Settings > App Appearance page ([#366](https://github.com/mabumusa1/facilities-management/pull/366))
- Settings — regional configuration: set company-wide default currency, locale (EN/AR), date format, timezone, and working-day schedule; values persisted in rf_system_settings and consumed by SLA tracking, Facilities availability, and document generation ([#366](https://github.com/mabumusa1/facilities-management/pull/366))
- Settings — notification preferences: enable/disable email and SMS per trigger (Leasing, Accounting, Service Requests, Facilities, Visitor Access) with bilingual message template overrides; persistent per-tenant configuration in rf_notification_preferences ([#366](https://github.com/mabumusa1/facilities-management/pull/366))
- Settings — audit trail: every settings change (invoice prefix, VAT rate, notification toggle, etc.) is logged in rf_settings_audit_logs with actor, setting group, key, old value, new value, and timestamp; paginated list viewable by admins ([#366](https://github.com/mabumusa1/facilities-management/pull/366))
- Properties — unit state machine: valid status transitions (available → under_maintenance / occupied / off_plan) enforced server-side; automated transition to occupied on lease activation; full audit trail in rf_unit_status_history tracking who changed status, when, and why; bulk status change validates each unit through the state machine individually ([#363](https://github.com/mabumusa1/facilities-management/pull/363))
- Properties — photo management per unit: upload, reorder, set primary, and delete photos; new Media sort_order and is_primary columns; first uploaded photo auto-set as primary; primary status auto-promotes to next photo on deletion; file type validation (jpg/jpeg/png/webp, 5 MB max) ([#364](https://github.com/mabumusa1/facilities-management/pull/364))
- Properties — bulk unit management: change status on multiple units in one operation via the state machine; export filtered unit list to Excel (XLSX) with community, building, category, and search filters; enhanced global search across the unit portfolio ([#364](https://github.com/mabumusa1/facilities-management/pull/364))
- Properties — building metadata management: edit number of floors, construction year, and upload building documents (permits, certificates); floor validation prevents lowering building floors below the highest floor occupied by existing units ([#363](https://github.com/mabumusa1/facilities-management/pull/363))
