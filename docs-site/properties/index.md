# Properties Overview

Properties are organized in a three-level hierarchy:

```
Community
└── Building
    └── Unit
```

This hierarchy allows you to manage large portfolios with multiple complexes, each containing multiple buildings and hundreds of units.

## Quick Navigation

| Section | Description |
|---------|-------------|
| [Communities](/properties/communities) | Top-level property groupings |
| [Buildings](/properties/buildings) | Physical structures within communities |
| [Units](/properties/units) | Individual rentable or sellable spaces |
| [Facilities](/properties/facilities) | Shared amenities within communities |

## Accessing Properties

Go to **Properties** in the left navigation menu. You can browse communities, drill down into buildings, and view individual units.

## Key Concepts

### Unit Statuses

| Status | Meaning |
|--------|---------|
| **Available** | The unit is empty and ready to lease or sell |
| **Rented** | The unit has an active lease |
| **Sold** | The unit has been sold |
| **Sold & Rented** | The unit was sold but the buyer is also renting it to a tenant |

### Marketplace

Units can be listed on the built-in marketplace for rental or sale. Potential buyers and tenants can express interest, schedule visits, and submit offers directly through the system.
