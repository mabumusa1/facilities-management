# Visitor Access

The **Visitor Access** system allows residents to register visitors and request entry permits for their guests.

## Visitor Access Workflow

```
Submitted (New) → Pending → Approved → Checked In → Checked Out
                          → Rejected
                → Canceled
```

## Requesting Visitor Access

Tenants and owners can submit visitor access requests for their guests.

1. Navigate to **More → Visitor Access** or **Dashboard → Visitors**
2. Click **+ New Visitor Request**
3. Fill in:

| Field | Description |
|-------|-------------|
| **Visitor Name** | Full name of the visitor |
| **Visitor Phone** | Contact number |
| **ID Number** | National ID or passport number |
| **Entry Date** | Expected visit date |
| **Entry Time** | Expected arrival time |
| **Exit Time** | Expected departure time (optional) |
| **Purpose** | Reason for the visit |
| **Unit** | Your unit number |

4. Click **Submit**

## Visitor Access Status

| Status | Meaning |
|--------|---------|
| **New** | Just submitted |
| **Pending** | Under review |
| **Approved** | Access granted; visitor can enter |
| **Rejected** | Access denied |
| **Checked In** | Visitor has entered the property |
| **Checked Out** | Visitor has departed |
| **Canceled** | Request was canceled |

## Approving Visitor Requests (Managers)

1. Navigate to **Requests → Visitor Access**
2. Open a **Pending** request
3. Review the visitor's details
4. Click **Approve** or **Reject**
5. Add a note if needed

The resident and gate security are notified of approved visits.

## Check-In and Check-Out

When an approved visitor arrives:
1. Open the visitor access record
2. Click **Check In**

When the visitor leaves:
1. Open the visitor access record
2. Click **Check Out**

This creates a full audit trail of all visits.

## Visitor Access History

Navigate to **More → Visitor Access History** to see a full log of all past and upcoming visitor access requests for your unit.

Admins can view the history across all units from **Dashboard → Visits**.
