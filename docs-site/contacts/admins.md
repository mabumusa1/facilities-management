# Admins & Staff

**Admins** are internal team members with access to the management platform. Each admin is assigned a role that determines what they can see and do.

## Viewing Admins

Navigate to **Contacts → Admins** to see all staff members.

## Adding an Admin

::: info
Only users with the **Admin** role can add new admin accounts.
:::

1. Navigate to **Contacts → Admins → + New Admin**
2. Fill in personal details (name, email, phone)
3. Select the **Manager Role**:
   - Admin
   - Accounting Manager
   - Service Manager
   - Marketing Manager
   - Sales & Leasing Manager
4. For **Service Managers**, select the **Service Type(s)** they will handle
5. Configure **Access Scope**:
   - **All Communities** – unrestricted access
   - **Specific Communities** – select from the list
   - **Specific Buildings** – select from the list
6. Click **Save**

The new admin receives an email invitation to set up their account.

## Admin Roles Reference

| Role | Access |
|------|--------|
| **Admin** | Full system access |
| **Accounting Manager** | Transactions and financial reports |
| **Service Manager** | Service requests (filtered by type) |
| **Marketing Manager** | Announcements and marketplace |
| **Sales & Leasing Manager** | Properties, leases, and marketplace |

See [Roles & Permissions](/guide/roles) for the complete capabilities matrix.

## Editing an Admin

Open the admin record and click **Edit**. You can update:
- Personal information
- Manager role
- Access scope (communities/buildings)

::: warning
Changing a manager's role immediately affects their access. Notify the user of any changes.
:::

## Deactivating an Admin

To remove a team member's access without deleting their record:
1. Open the admin record
2. Click **Edit**
3. Set their **Status** to **Inactive**
4. Save

They will no longer be able to log in.
