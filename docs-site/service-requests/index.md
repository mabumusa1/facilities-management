# Service Requests Overview

The Service Requests module manages maintenance, repairs, and other service needs across your properties. It connects residents with service professionals through a structured workflow with full status tracking.

## Request Types

| Type | Description |
|------|-------------|
| **Home Service Requests** | Maintenance and repairs inside individual units (plumbing, electrical, HVAC, etc.) |
| **Common Area Requests** | Issues in shared spaces (lobby, pool, gym, parking, etc.) |
| **Visitor Access Requests** | Entry permits for visitors (managed via [Visitor Access](/facilities/visitor-access)) |
| **Facility Booking Requests** | Reservations for community facilities (managed via [Facility Bookings](/facilities/bookings)) |

## Request Lifecycle

```
New → Assigned → Accepted → In Progress → Invoice Created → Invoice Accepted → Resolved
              → Rejected
                          → Quote Sent → Quote Accepted → Invoice Created
                                       → Quote Rejected → Rejected
                          → Rescheduled
→ Canceled (at any point before resolution)
```

## Accessing Service Requests

Navigate to **Requests** in the left menu. The list view shows all requests with quick filters:
- **Status** (All, New, Assigned, In Progress, Resolved, Canceled)
- **Type** (Home, Common Area)
- **Community / Building / Unit**
- **Date range**
- **Assigned professional**
